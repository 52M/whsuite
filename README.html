System Requirements

In general, WHSuite will work with most good Linux based web hosting providers offering up to date PHP hosting.

WHSuite does not currently officialy support Windows hosting. If you are interested on using WHSuite on a Windows based hosting environment, please do visit our forums for any discussions about doing so.

WHSuite's fully system requirements can be found on our documentation website at docs.whsuite.com.

Installation

Please install into a fresh directory and empty database.

To get started with the installation, upload the provided files to your hosting account using an FTP client. The following files/directorys should have write access for your immediate user:

  • /whsuite/app/storage (recursive)
  • /whsuite/app/configs

Once you've set your permissions, create your MySQL database. It's recommended that your database collation/encoding is be set to "utf8_general_ci". Next, browse to http://<your-whsuite-domain>/install/ to start the installation. From there, follow the on screen instructions

.htaccess File

WHSuite supports running both in a sub directory (e.g yourhost.com/billing) or a subdomain (eg billing.yourhost.com) however on some server environments, you'll need to add a line to your .htaccess file in the root of your installation.

If when you first browse to the install directory after uploading WSuite you see no styling, you'll need to make this minor modification.

Open your root .htaccess file (this is located in the same place as your index.php and whsuite directory) with a text editor of your choice.

Near the top you'll see a line that reads #RewriteBase /. Uncomment this line.

If you're working in a sub-directory, you'll want this line to include that directory name. So if you were installing at yourhosting.com/billing this line should read:

RewriteBase /billing/

When your installation is finished we strongly recommend deleting the install.php file from your server.

Upgrading

To upgrade your installation from a previous version of WHSuite, simply replace the primary WHSuite files, leaving in place any custom uploads or other site-specific files. These are stored in your /whsuite/app/storage/ directory. Remember to also back up any custom changes you've made to the default themes.

When replacing the files, you should choose the 'merge' option in your FTP program to retain any additional files such as your database configuration file located in /whsuite/app/config

The following files should be kept in tact from your previous installation:

  • /whsuite/app/config/database.php
  • /whsuite/app/storage/whsuite.installed
  • /whsuite/app/storage/uploader (if present)

Once you've uploaded the modified files, browse to http://YOUR-SITE/install to upgrade your installation. Follow the on-screen instructions to complete the upgrade process.

When your upgrade is finished we strongly recommend deleting the install.php file from your server.

Updating Addons

After upgrading WHSuite, visit your admin area addons page to check if there are any updates available. If you wish to apply these updates, simply click the update button for each addon.

Support

If you have a problem with a specific addon, please create an issue on that addons repository. You can find all our repositories here.
